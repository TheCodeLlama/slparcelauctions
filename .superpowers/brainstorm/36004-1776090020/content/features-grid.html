Tradeoffs

Option A — Bento

  • Matches Stitch reference & "intentional asymmetry" from DESIGN.md §1
  • Natural visual hierarchy — the 3 large cards draw the eye first, small cards fill in
  • Visually "breathed into existence" per the Digital Curator aesthetic
  • One FeatureCard component with a size="sm" | "lg" variant prop
  • Real-Time Bidding + Verified Listings can use the decorative bidding-bg-light.png image + dark-surface variant respectively

Option B — Uniform Grid

  • Simpler mental model — one card size, clean CSS
  • Easier to add/remove features later without re-balancing the bento
  • Better responsive collapse — grid just stacks on mobile
  • Bends the "intentional asymmetry" rule — feels more template-y
  • Loses the editorial-first composition the spec explicitly asks for
A

Asymmetric bento (recommended)

Three 2-span large cards alternating with three 1-span small cards. Matches Stitch reference. FeatureCard supports size="sm" | "lg" + variant="surface" | "primary" | "dark".

B

Uniform 3×2 grid

Six identical cards. Simpler, but loses the editorial composition.
